Setting up InfluxDB in a container

Note: This is a distillation of the InfluxDB setup documentation that I used - it can be found HERE

Assuming you’re already set up with a nice docker host somewhere, run the following, but change the variable values:

docker run --rm influxdb influxd config > influxdb.conf

followed by

docker run --rm \
-e INFLUXDB_DB=db0 \
-e INFLUXDB_ADMIN_ENABLED=true \
-e INFLUXDB_ADMIN_USER=admin \
-e INFLUXDB_ADMIN_USER=supersecret \
-e INFLUXDB_USER=telegraf \
-e INFLUXDB_USER_PASSWORD=supersecret \
-d -p 8086:8086 \
-v $PWD/influxdb.conf:/etc/influxdb/influxdb.conf:ro influxdb -config /etc/influxdb/influxdb.conf

Prometheus in a container

Following the instructions HERE will get you up and running with Prometheus in a Docker container.

In summary, paste this into a terminal, and you’ll be up and running almost instantly.

echo "
global:
  scrape_interval:     15s
  evaluation_interval: 15s
  external_labels:
      monitor: 'my-monitor'
scrape_configs:
  - job_name: 'prometheus'
    static_configs:
      - targets: ['192.168.111.55:9090', '192.168.111.50:9121', '192.168.111.55:9273']
" > prometheus.yml

You may need to add more data sources (targets) to the configuration file later and restart prometheus after updating the prometheus.yml file.

Start the container:

docker run -p 9090:9090 -v $PWD/prometheus.yml:/etc/prometheus/prometheus.yml:ro \
  -d prom/prometheus -config.file /etc/prometheus/prometheus.yml

Prometheus, once started, will begin scraping any prometheus client endpoints you’ve declared as a target within its configuration file.

To view prometheus’ data inputs (and search it, if desired), browse to the IP of your prometheus instance at port 9090 (http://localhost:9090).

Grafana in a container

Grafana provides a beautifully designed interface for displaying real-time information pulled from many different time-series data sources. It is able to ingest from Prometheus, InfluxDB, New Relic, PRTG, and many, many more.

Grafana provides a nice (albeit somewhat small) searchable index of some pre-made dashboards. The ones I’ve tried have been excellent - most notably the Redis exporter for prometheus, and SQL Server via Telegraf dashboards.

To stand up a Grafana container, use the below snippet (taken from HERE:

docker run -d -p 3000:3000 \
    -v /var/lib/grafana:/var/lib/grafana \
    -e "GF_SECURITY_ADMIN_PASSWORD=secret" \
    grafana/grafana

You will now be able to connect to the Grafana frontend via http://dockerhost:3000/.

Configuring Telegraf

Telegraf is an agent used to monitor servers of various sorts. It provides plugins for a number of inputs, and plugins for a number of output formats as well. Included among the outputs are Prometheus, and the default, InfluxDB.

The Telegraf agent can be configured via its telegraf.conf file. Below, the example configuration will set up monitoring for MariaDB, and RabbitMQ. We will use the Redis Exporter app to monitor that application. To create a base configuration, run the below command, or to make your own from the Telegraf config generator, run telegraf config > telegraf.conf:

echo '
[[outputs.prometheus_client]]
  listen = "0.0.0.0:9273"
  # path = "/metrics"
  expiration_interval = "60s"

[[outputs.influxdb]]
  urls = ["http://192.168.111.55:8086"]
  database = "db0"
  retention_policy = ""
  write_consistency = "any"
  timeout = "5s"
  username = "telegraf"
  password = "supersecret"

[[inputs.rabbitmq]]
  url = "http://192.168.111.50:15672"
  name = "octo-pi"
  username = "telegraf"
  password = "telegraf"

[[inputs.mysql]]
  servers = ["telegraf:telegraf@(127.0.0.1:3306)/"]
  perf_events_statements_digest_text_limit  = 120
  perf_events_statements_limit              = 250
  perf_events_statements_time_limit         = 86400
  table_schema_databases                    = []
  gather_process_list                       = true
  gather_user_statistics                    = true
  gather_info_schema_auto_inc               = true
  gather_innodb_metrics                     = true
  gather_slave_status                       = true
  interval_slow                             = "30m"

' > telegraf.conf

Bringing it all together (the fun part)

This effort (which be turned into a simple bash script) results in your ability to make very clean and versatile dashboards, complete with the ability to query data from the available inputs!

Once data is viewable in the Prometheus management interface, add data inputs for the newly created Prometheus container and InfluxDB server in Grafana. Then, create new dashboards using the Grafana’s premade dashboard search site. Copy the ID of the dashboard you wish to use (and for which you have data inputs available for) into the form created when clicking “Import Dashboard” from the “Dashboards” menu in Grafana (screenshot below).
